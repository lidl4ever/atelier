# Atelier — 開發計畫
> 目標：做一個比 WeTransfer Paper 更好、更直覺的 iPad 繪畫 app。全功能免費，無付費牆。

## 一、WeTransfer Paper 研究摘要

**它的好**
- 五種手調筆刷：鉛筆（Sketch）、鋼筆（Write）、麥克筆（Outline）、墨水筆（Draw）、水彩（Color），手感極佳
- 全手勢介面：幾乎無按鈕。捏合縮放、兩指逆時針旋轉「Rewind」復原
- 日記本（Journal）式作品管理，沉浸、低門檻
- Apple Pencil 壓感支援

**它的痛（我們的機會）**
- 被收購後 Pro 訂閱約 **$70/年**；調筆刷大小、跨頁剪貼等基本功能都鎖在付費牆
- **沒有圖層系統**（Procreate 有、Paper 一直沒有）
- 匯出選項有限、解析度受限

## 二、產品定位

| | Paper | Atelier（本案） |
|---|---|---|
| 價格 | $70/年 | 完全免費 |
| 圖層 | ✗ | ✓ 多圖層＋透明度＋合併 |
| 筆刷大小調整 | 付費 | 免費 |
| 手勢 | Rewind | 兩指點＝復原、三指點＝重做、捏合＝縮放平移 |
| Pencil | 壓感 | 壓感＋傾斜（鉛筆側鋒） |
| 資料 | 雲端綁定 | 全部留在裝置（IndexedDB） |

## 三、技術架構（為日後轉 Swift 而設計）

單一 `index.html`，三層分離：

1. **繪圖引擎**（純邏輯）：筆刷演算法（取樣→平滑→沿路徑蓋章/描線段）、圖層合成、復原堆疊 → 日後 1:1 對照移植到 Swift/Metal
2. **輸入層**：Pointer Events（壓感 `pressure`、傾斜 `tiltX/Y`、高頻取樣 `getCoalescedEvents`）→ Swift 對應 `UITouch.force/altitudeAngle/coalescedTouches`
3. **UI 層**：HTML/CSS 極簡浮動工具列 → 對應 SwiftUI

**Swift 移植對照**：Canvas 2D → PencilKit 或 Metal；IndexedDB → Core Data/檔案；Pointer Events → UIKit touches。引擎邏輯（本案最有價值部分）可直接翻譯。

## 四、功能規格

- **筆刷 ×5**：鉛筆（顆粒質感、傾斜變側鋒）、墨水筆（壓感粗細）、麥克筆（multiply 疊色）、水彩（低流量暈染疊色）、橡皮擦
- **圖層**：新增/刪除/隱藏/透明度/向下合併，縮圖即時預覽
- **手勢**：兩指捏合縮放平移、兩指輕點復原、三指輕點重做；偵測到 Pencil 後自動進入「手指只做手勢」模式（防手掌誤觸）
- **作品集**：自動儲存（IndexedDB）、縮圖牆、重新命名、刪除
- **匯出**：全解析度 PNG
- **介面**：極簡沉浸式——畫布全螢幕、底部浮動工具膠囊、下筆時介面自動淡出

## 六、v1.1 更新（2026-06-10）

- **鉛筆引擎重寫**：「紙紋錨定」石墨模擬——顆粒來自一張無縫紙紋（value noise），所有筆觸從同一張紙取樣，質感連續自然；壓力決定石墨壓進紙齒深淺（三段 threshold band）
- **筆芯硬度 3H–9B＋炭筆**：選鉛筆時自動出現筆芯列；炭筆用較粗紙紋、軟邊
- **傾斜側鋒**：Pencil 傾斜 → 筆觸變寬變淡（畫陰影）
- **手指＝塗抹模式**：手指在畫布上拖動會把顏料推開（指腹暈染）；繪圖用 Pencil
- **各工具獨立大小**：橡皮擦、塗抹、每支筆刷各自記憶大小

## 七、v1.2 更新（2026-06-10）— 真實質感研究後重調

**鉛筆（研究結論 → 實作）**
- 真實筆觸邊緣是「硬而破碎的毛邊」不是柔焦 → 邊緣遮罩改硬（inner .62），毛邊交給紙紋與拖痕
- 石墨沿運筆方向有刮痕 → 新增各向異性拖痕遮罩（destination-out），以筆畫弧長錨定、沿方向連續
- 硬筆芯物理上無法變黑 → 每級筆芯有明度上限；重壓仍保留紙齒顆粒
- 紙峰先沾墨 → 紙紋雜訊 gamma 銳化＋三段壓力 band 重新校準

**毛筆（生宣／熟宣研究 → 實作）**
- 提按：筆鋒髮絲 → 筆腹全寬（壓力 1.6 次方曲線）
- 一筆內墨量遞減；墨量低於門檻 → 飛白（沿筆方向白絲）
- 暈染強度由紙張吸水性決定：熟宣不暈、素描紙微暈、生宣明顯暈開

**紙張系統**
- 紙色 ×4（米白／純白／暖灰／牛皮）、紙紋 ×3（細滑／中目／粗目）
- 紙紋同時影響：鉛筆顆粒對比與尺度、毛筆暈染半徑

## 八、v1.3 更新（2026-06-11）— 毛筆深度重寫

依書法筆法研究（一筆三段、五墨、MoXi 類疊印模型）重寫毛筆：

- **一筆三段**：起筆露鋒（入筆收窄）→ 行筆 → 收筆自動出鋒（沿最後方向收尖，解決圓鈍筆尾）
- **提按主控寬度**（1.8 次方曲線：筆鋒髮絲 → 筆腹全寬），疾筆變細
- **五墨**：墨量一筆內遞減；墨乾或真正疾筆 → 飛白。飛白改在收筆時「整筆分段擦出」
  （蓋章逐章擦痕會被重疊抵銷——這是教訓），段以弧長錨定、精確相接
- **暈染重寫**：不再是高斯光暈——縮放模糊（雙層擴散）＋整張紙纖維遮罩
  一次性 destination-in（逐磚套用會只剩最後一磚，已修正），只有濕筆才暈、吸水紙才暈
- **積墨**：multiply 合成，交疊筆畫自然變濃

## 九、v1.5 更新（2026-06-11）— 整合 libmypaint 專業筆刷引擎

- 鉛筆改用 **MyPaint 筆刷引擎**（libmypaint.js，ISC 授權）＋ **David Revoy（deevad）專業鉛筆筆刷**（CC0）——與 MyPaint/GIMP 同款引擎
- 混合架構：MyPaint 負責筆觸動態（dab 疊印、壓力曲線、抖動），Atelier 紙紋系統負責紙齒質感（destination-in 整張遮罩）
- 筆芯 3H–9B/炭筆 → 映射至 4H/2B 基底筆刷＋濃度/半徑參數
- 側鋒改為筆畫層級（起筆傾角配置本筆寬度/濃度；此編譯版不支援逐點 tilt）
- 效能：0.02ms/取樣點（比自製引擎快 26 倍）；舊引擎保留為後備
- 教訓：MyPaint declination 軸 90°=直立；stroke_to 的 tilt 參數範圍 -1..1 非角度

## 十一、GPU 鉛筆 g4 — 寬度校準（2026-06-22）

獨立測試頁 `test-pencil-g4.html`（線上：https://lidl4ever.github.io/atelier/test-pencil-g4.html ），baked-grain 顆粒模型。筆寬以真實毫米校準（iPad ≈264ppi）。

**寬度規格（直徑，校準目標）**

| 狀態 | 直徑 |
|---|---|
| 直立尖端 | 0.33–0.77mm（隨壓力） |
| 中等傾斜 | ~8mm |
| 放斜 | ~14mm |
| 幾乎躺平 | ~24mm |

- 公式（v5）：`widthMM = 尖端直徑 + TILT_BROAD_MM × tilt^TILT_CURVE`，`tilt = (90 − 高度角) / (90 − ALT_FLAT_DEG)`（clamp 0..1）
- 旋鈕：`NIB_MIN_MM 0.32` / `NIB_MAX_MM 0.77`（尖端壓感範圍）、`TILT_BROAD_MM 23.5`（躺平鋪寬 → ~24mm）、`ALT_FLAT_DEG 25`（高度角 ≤ 此值＝完全躺平，越大越早變寬）、`TILT_CURVE 1.5`（鋪寬曲線，越小中段越早變寬）
- v6 實機驗證：`altitudeAngle` 正常運作（HUD 顯示 `alt` 非後備）；筆貼平實測高度角可低至 14°，tilt 滿格、寬度達設計上限。v7 依需求把躺平上限拉到 ~24mm。

**🔒 已鎖定（v7，勿動）— 斜度與粗度**
| 參數 | 值 |
|---|---|
| `NIB_MIN_MM` / `NIB_MAX_MM` | 0.32 / 0.77（尖端壓感直徑 mm） |
| `TILT_BROAD_MM` | 23.5（躺平側鋒額外鋪寬 → ~24mm） |
| `ALT_FLAT_DEG` | 25（高度角 ≤ 此值＝完全躺平） |
| `TILT_CURVE` | 1.5（鋪寬曲線） |
| `PX_PER_MM` | 132×DPR/25.4（≈10.4 @DPR2/264ppi） |

對照寬度：直立 ~0.5mm｜中等(60°) ~8mm｜放斜(50°) ~12mm｜躺平 ~24mm。

### v8（2026-06-22）— 邊緣質感
- 問題：傾斜側鋒邊緣發散（像 airbrush 暈圈），dab 剖面是高斯（`soft` 傾斜時達 0.94，整顆從半徑 6% 就淡出）。
- 目標：Paper 那種「平頂＋窄邊、邊界乾淨利落，邊緣以顆粒碎開」。
- 改法：新增 `EDGE_SOFT_TIP 0.22` / `EDGE_SOFT_FLAT 0.34`（取代寫死的 0.30→0.94），把傾斜邊緣大幅收窄成平頂；邊緣顆粒碎裂續由 `EDGE_BITE 0.60` 負責。

### v9（2026-06-22）— 不透明度 + 近直立太粗 + azimuth 診斷
- 不透明度：單筆太實、疊加看不出層次 → 新增 `STROKE_OPACITY 0.55`（cov 整體乘係數），單筆半透明、疊加才變深（真鉛筆感）。
- 近直立太粗：`TILT_BROAD_MM` 23.5 太大、曲線只有 1.5，稍微一斜（70°）就 4mm。`TILT_CURVE` 1.5 → 2.0，把 broaden 集中到躺平端，近直立維持細（真直立 90° 仍 0.43mm；80° ~1mm；躺平仍 24mm）。⚠ 等於微調了 v7 鎖定的曲線（保留 24mm 上限）。
- 漸層（待做）：Paper 側鋒帶有「筆芯側深→另一側淡」斜向漸層，需 `azimuthAngle`。v9 先把 azimuth 加進 HUD 確認裝置有回報，v10 再實作跨筆觸漸層。

### v10（2026-06-22）— 細線平台區
- `azimuthAngle` 確認可用（HUD 實測 `azi -52°`），漸層改排到後續版本。
- 細線不該只在 90°：自然握姿本來就帶傾斜。新增 `ALT_TIP_DEG 60` — 高度角 ≥ 60° 都維持最細 0.4mm（細線平台），60°→25° 才鋪寬。tilt 正規化改為 `(ALT_TIP_DEG−alt)/(ALT_TIP_DEG−ALT_FLAT_DEG)`。
- 對照：90~60° 0.4mm（平台）｜50° ~2.4mm｜40° ~8mm｜躺平 24mm。

### v11（2026-06-22）— 加成疊印 + 斜向漸層 + 平台延伸
- 平台 `ALT_TIP_DEG` 60 → 50（自然握姿更斜也維持細線）。
- 重疊可疊深：dab blend 由 `MAX`（壓平、無層次）改為**加成式 over 合成**（premultiplied `ONE, ONE_MINUS_SRC_ALPHA`）。新增 `DAB_FLOW 0.10`（每 dab 沉積量）：單筆半透明、重疊處自然變深。紙紋錨定（`vWorld` 取樣）使重疊不會糊化顆粒。間距 `r*0.28` → 每點約 7 dab。
- 斜向漸層（筆芯側深→另一側淡）：新增 per-instance `aAzi`（azimuthAngle，方向向量內插避免 ±π 接縫），FS 沿筆方位做 `lateral` 漸層，`EDGE_GRAD 0.35`（另一側濃度比例），僅傾斜時生效。
- ⚠ 待實機驗證：`DAB_FLOW` 整體濃淡、`EDGE_GRAD` 對比、漸層**方向是否需反向**（azimuth 符號）。

### v12（2026-06-24）— 修串珠/節點 artifact
- 問題：v11 加成模型下，快速細線出現週期性圓點、傾斜寬筆觸出現大圓點感。
- 根因：dab 間距（`max(0.8, r*0.28)`）≈ dab 邊緣寬度，加成下每顆中心較濃 → 沿線週期凸起串珠（MAX 模式會壓平故無此問題）。
- 改法：間距收緊為 `max(0.35, r*0.12)`（與半徑成比例 → 每點覆蓋 dab 數固定 ~16，濃淡與尺寸無關且夠密融合）；`DAB_FLOW` 0.10 → 0.045 補償密度，維持單筆半透明＋重疊疊深。
- v13：細筆尖（半徑 ~1.2–2px）下限 0.35 仍≈邊緣寬度、殘留串珠 → 下限降到 `0.2`，最細尖也夠密融合。
- 教訓 v4：`hypot(tiltX,tiltY)` 跟真實傾斜角非線性對應，中等傾斜嚴重低估 → 中段吃不到鋪寬，只有快貼螢幕才暴衝變寬。
- 教訓 v5：改用 Pencil 直接回報的 **`altitudeAngle`（真實高度角）** 當傾斜訊號（tiltX/Y 僅作後備），中等傾斜即線性反映寬度；曲線放緩為 `tilt^1.5`。中等傾斜 ~4.5mm、放斜 ~7mm、躺平 ~10mm。
